# v97 - 빈 표 검증 오판정 수정 (연관 에러 / 설계 횡전개 대상)

## 문제

'연관 에러' 표와 '설계 횡전개 대상' 표가 **데이터 없이 비어 있는데도** 검증 결과가
`사유 작성`(= 정상)으로 잘못 처리되었다.

## 원인

`_has_reason_text_around_table()`의 HTML/Wiki 문자열 파싱 경로에서 **HTML 태그를 먼저
통째로 제거**한 뒤 표를 제거하려고 했다. 그 결과 `<table>` 태그가 먼저 사라져 표 헤더/셀
텍스트(예: `에러명`, `에러 조건`, `설비 유형`, `모델`, `라인`, `Jira Project`)가 본문
텍스트로 남았고, 이 표 헤더 텍스트가 '사유 텍스트'로 오인되어 빈 표인데도 `사유 작성`으로
판정되었다.

Jira wiki markup(`||헤더||`, `|셀|`) 형태의 표 역시 제거되지 않아 동일하게 오판정되었다.

## 수정

문자열(HTML/Wiki) 경로에서 **표 내용을 가장 먼저** 제거하도록 순서를 바로잡았다.

- HTML 표: `<table>...</table>` 블록을 태그 제거 이전에 삭제
- Wiki markup 표: `|` 로 시작하는 표 행 라인을 삭제

이후 남은 텍스트(소제목 옆 부연 텍스트, 표 위/아래 부연 설명 문단)에 대해서만 사유 여부를
판단한다.

## 판정 기준 (요구사항)

빈 칸(표에 데이터 없음)일 경우:

- 소제목('연관 에러' / '설계 횡전개 대상')만 있고 표 헤더 외 아무 텍스트가 없으면 → `FAIL`
- 소제목 옆 또는 표 위/아래에 부연 설명 글(사유)이 있으면 → `사유 작성`
- 표에 실제 데이터가 있으면 → `OK`

## 검증

- HTML/Wiki 빈 표 → `FAIL` (기존: `사유 작성` 오판정)
- HTML/Wiki + 부연 설명(사유) → `사유 작성`
- 표 데이터 존재 → `OK`
- ADF(JSON) 경로는 기존과 동일하게 정상 동작 (회귀 없음)
