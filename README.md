# SCCB Automation

Jira와 Confluence를 연동하여 SCCB(소프트웨어 변경 통제 회의) 업무를 자동화하는 데스크톱 GUI 도구입니다.

<img width="1495" height="903" alt="1" src="https://github.com/user-attachments/assets/0096ee32-0fcc-4418-a37d-79b31511b4b9" />

## 실행

```bash
pip install -r requirements.txt
python -m sccb_app.app
```


---
## Python 없이 실행하기

Python이 설치되어 있지 않은 사용자는 저장소의 `releases/` 폴더에 있는 빌드된 실행파일을 사용할 수 있습니다.

- Windows: build.bat 실행 후 `releases/JiraSCCB.exe`를 더블클릭해서 실행합니다.
- Linux: `releases/JiraSCCB-linux`
  - 실행 권한이 없으면 `chmod +x releases/JiraSCCB-linux`를 먼저 실행하세요.

실행파일 보관 및 갱신 절차는 `releases/README.md`를 참고하세요.

## 주요 기능

### 1. 접속 및 로그인

- Jira 서버 URL, ID/PW를 입력하고 **Login 확인** 버튼(또는 Enter)으로 연결 상태를 확인합니다.
- SSL 검증 옵션과 최대 조회 건수(Max Results)를 설정할 수 있습니다.

---

### 2. 이슈 조회 (JQL 검색)

JQL 입력란에 직접 쿼리를 입력하거나, 아래의 프리셋 버튼을 클릭해 빠르게 조회합니다.

| 버튼 | 설명 |
|------|------|
| **SCCB 미대상** | SCCB 상태가 "미 대상"인 이슈 조회 |
| **SCCB 대상** | SCCB 대상 이슈 조회 (빈칸·완료·사전완료 포함) |
| **VOC 완료처리** | 이번주 SCCB URL의 이슈에 연결된 SW_VOC 이슈 조회 |

- 결과 그리드의 **열 헤더를 클릭**하면 오름차순/내림차순 정렬이 적용됩니다.
- **이번주 SCCB URL**을 입력해 두면 조회 결과에서 주간 SCCB 이슈가 **녹색**으로 강조 표시됩니다.

---

### 3. SCCB 대상 검증 (자동 항목 점검)

**SCCB 대상** 모드로 조회하면 각 이슈에 대해 아래 항목을 병렬로 자동 점검합니다.

| 열 | 내용 |
|----|------|
| 본문 길이 | Jira 이슈 본문의 글자 수 (300자 미만이면 빨간색 표시) |
| 설계횡전개 | 설계 횡전개 표 포함 여부 (OK / FAIL) |
| 연관 에러 | 연관 에러 테이블 포함 여부 (OK / FAIL) |
| 이슈연결 상태 | 이슈 링크 필수 항목 누락 여부 (OK / Missing: ...) |
| LLM TC 생성 | LLM 기반 테스트케이스 생성 여부 (OK / FAIL) |
| AIO Test | AIO 테스트 실행 결과 (OK(통과/전체) / FAIL) |
| P/R 병합 | Pull Request 병합 상태 (MERGED / OPEN / NONE) |

---

### 4. 이슈 상태 전이 (일괄 처리)

체크박스로 이슈를 선택한 뒤 아래 버튼으로 상태를 일괄 변경합니다.

| 버튼 | 동작 |
|------|------|
| **선택 이슈 Approval** | In Verification → Approval 전이 |
| **선택 이슈 Complete** | In Verification → Approval → Complete 순차 전이 |

- **SCCB 대상** 모드: Complete 전이 시 `SCCB 완료일`·`사전 SCCB 의뢰일` 날짜를 자동 입력합니다.
- **SCCB 미대상** 모드: Complete 전이 시 위 날짜 필드를 비워둡니다.

---

### 5. VOC 완료처리

1. **이번주 SCCB URL**을 입력합니다.
2. **VOC 완료처리** 버튼을 클릭하면 해당 페이지의 이슈에 연결된 **SW_VOC 이슈** 목록을 조회합니다.
3. **선택 이슈 Complete**를 누르면 연결된 SW_VOC 이슈를 완료 처리합니다.
   - `Start Date`는 완료일 기준 13–15일 전 날짜 중 랜덤 적용, `End Date`는 처리 당일로 자동 설정됩니다.

---

### 6. Confluence 페이지 자동 생성

#### SCCB 주간 페이지 복제

**SCCB Page 생성** 버튼을 클릭하면 **이번주 SCCB URL**의 페이지를 Storage 포맷 그대로 복제하여 **다음 주 페이지**를 생성합니다.

- 표, 셀 병합, 글꼴/색상, Jira 매크로, 페이지 레이아웃 등 원본 본문 구조를 그대로 복제합니다.
- 주간 기간은 월요일~일요일 기준으로 자동 산정합니다.
  - 예: `26W (06/22~06/29)` → `27W (06/29~07/05)`
- `사전SCCB 검토 의견`의 Handy Date 시작일/종료일도 새 주 범위로 자동 갱신합니다.
- 연도가 바뀌면 `YY년 사전 SCCB 기록` 상위 페이지를 생성하거나 재사용하고, 그 아래에서 `1W`부터 시작합니다.
- 동일 상위 페이지에 같은 주차가 이미 있으면 중복 생성하지 않고 기존 URL을 반환합니다.
- 생성 완료 후 URL 입력칸이 새 주 페이지 URL로 자동 변경되고 브라우저로 열립니다.

#### SCCB 회의록 페이지 복제

**회의록 Page 생성** 버튼을 클릭하면 원본 회의록 URL 입력 팝업이 열립니다.

- 원본 형식(표/레이아웃/Jira 매크로)을 Storage 포맷으로 복제합니다.
- 제목의 회의일과 `날짜/시간` 표 날짜를 다음 주로 자동 이동합니다.
- `SCCB 리스트` Jira 매크로의 `SCCB 완료일` 시작/종료 조건을 7일 이동합니다.
- 연도 변경 시 `YY년 회의록 (SCCB)` 상위 페이지를 생성하거나 재사용합니다.

---

### 7. Excel 내보내기

**Export Excel** 버튼을 클릭하면 현재 조회된 이슈 목록과 검증 결과를 `.xlsx` 파일로 저장합니다.

- `RESULT` 열: 모든 검증 항목을 종합한 최종 판정 (OK / FAIL / N/A)
- 자동 필터 및 첫 행 고정이 적용됩니다.
