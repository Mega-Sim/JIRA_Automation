## v93 — 매주 SCCB Confluence Page 포맷 복제

- `이번주 SCCB URL`에 현재 주간 Confluence 페이지 URL을 입력한 뒤 **SCCB Page 생성**을 누르면, 다음 주 페이지를 원본 주간 페이지의 **Confluence storage format**으로 복제합니다.
- 표, 셀 병합, 글꼴/색상, Jira 매크로, 페이지 레이아웃 등 원본 본문 구조를 그대로 사용합니다.
- 첨부파일이 없는 SCCB 페이지를 전제로 하며, 파일/이미지·댓글·이력·watcher·제한 권한은 복제하지 않습니다.
- 주간 기간은 월요일~일요일 기준입니다.
  - `26W (06/22~06/29)` → `27W (06/29~07/05)`
- 연도가 바뀌면 `YY년 사전 SCCB 기록` 상위 페이지를 만들거나 재사용하고, 그 아래에서 `1W`부터 시작합니다.
- 이미 같은 상위 페이지에 같은 주차가 있으면 중복 생성하지 않고 기존 페이지 URL을 사용합니다.
- 생성이 끝나면 URL 입력칸이 새 주 페이지 URL로 자동 변경됩니다.

상세 변경 사항은 `CHANGELOG_v93_storage_clone.md`를 참고하세요.

## v95: SCCB 회의록 다음 주 페이지 생성

`회의록 Page 생성`을 누르면 원본 회의록 주소를 입력하는 팝업이 열립니다.

- 원본 형식(표/레이아웃/Jira 매크로)을 Storage 본문으로 복제합니다.
- 제목의 회의일과 `날짜/시간` 표 날짜를 다음 주로 이동합니다.
- `3. SCCB 리스트` Jira 매크로에서 `SCCB 완료일` 시작/종료 조건을 7일 이동합니다.
- 연도 변경이면 `YY년 회의록 (SCCB)` 상위 페이지를 만들거나 재사용한 뒤 `1W`부터 만듭니다.
- 제목 끝의 `N건(A0, B3, C2)` 문구는 원본을 보존합니다. 생성 뒤 Jira 매크로가 새 JQL 기준으로 렌더링한 실제 목록을 확인해 필요하면 제목의 건수 문구만 수정합니다.

## v96: 회의록 URL 입력 팝업

- 메인 화면에는 **이번주 SCCB URL**만 남겼습니다.
- `회의록 Page 생성` 클릭 시 회의록 원본 URL 입력 팝업이 열립니다.
- URL 입력 후 **확인** 또는 **Enter**로 다음 주 회의록 생성이 시작됩니다.
- Esc, 취소, 창 닫기는 생성 작업을 취소합니다.
