> **v93에서 대체됨:** SEMES Confluence 9.2의 실제 복사 창은 `docopypage.action` HTML form 방식이 아니므로, 이 구현은 v93의 storage format 복제 방식으로 대체되었습니다.

# v92: SCCB Page 생성 — 이전 주 Confluence 페이지 복사 방식

## 변경 사항

### 1. 본문 재생성 대신 Confluence 기본 페이지 복사 사용
- **SCCB Page 생성** 버튼은 이전 주 페이지의 본문을 단순 REST `POST /content`로 새로 만드는 방식이 아니라, Confluence의 기본 페이지 복사 화면(`copypage.action` → `docopypage.action`)을 통해 다음 주 페이지를 만듭니다.
- 사내 Confluence Server/Data Center 계열에서는 단일 페이지 복사 공개 REST API가 제공되지 않는 버전이 있어, 실제 화면에서 사용하는 복사 흐름을 자동화했습니다.
- Confluence가 복사한 페이지에 대해 제목과 주간 날짜 범위만 후처리로 치환합니다.

### 2. 유지되는 항목
- 표 구조, 셀 병합, 글꼴, 색, 매크로, 페이지 레이아웃, 본문 HTML(Storage format)
- 페이지에 포함된 파일·이미지: 복사 화면에 첨부파일 포함 선택지가 있으면 자동으로 선택
- 단일 페이지 복사 방식이므로 **하위 페이지, 댓글, 이력, watcher 정보는 복제하지 않습니다.**
- 원본의 페이지 제한(Restriction) 복사 여부는 Confluence 권한 및 서버 정책을 따릅니다.

### 3. 기존 주차/연도 기능 유지
- 주간 날짜: 월요일~일요일 (`26W (06/22~06/29)` → `27W (06/29~07/05)`)
- 연도 전환: `27년 사전 SCCB 기록` 상위 페이지 생성 또는 재사용 후 `1W`부터 생성
- 동일 상위 페이지 아래에 같은 제목이 이미 있으면 중복 복사하지 않고 기존 페이지를 엽니다.
- 생성이 끝나면 URL 입력칸은 새 주 페이지 URL로 바뀝니다.

## 실제 사내 환경 확인 필요
- 사내 Confluence의 실제 복사 화면 HTML/권한 설정에 따라 첫 실행에서는 한 번의 실환경 검증이 필요합니다.
- 계정에는 대상 Space의 페이지 생성 권한과 원본 페이지 조회 권한이 있어야 합니다. 첨부파일을 함께 복사하는 경우에는 첨부파일 접근 권한도 필요합니다.
